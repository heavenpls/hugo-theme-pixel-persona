# Pixel Persona 开源发布说明

## 推荐仓库名称

推荐使用：

`hugo-theme-pixel-persona`

推荐理由：

- 符合 Hugo 主题仓库的常见命名习惯，辨识度高。
- 和当前主题内部名称 `pixel-persona` 保持一致。
- 便于在 GitHub、Hugo 主题目录和搜索引擎中被检索到。

## 可选仓库名称

- `pixel-persona-hugo-theme`
- `pixel-persona`
- `hugo-theme-pixel-persona-retro`

如果你希望优先突出 Hugo 主题属性，仍然建议首选 `hugo-theme-pixel-persona`。

## 中文主题名

建议中文展示名：

`像素人格`

这个名字和英文 `Pixel Persona` 一一对应，适合作为 README 标题、海报文案、发布介绍和截图封面文案。

## 仓库简介

### 中文简介

一款面向个人博客的像素风 Hugo 主题，强调长文阅读体验、复古像素美学、深色代码块与内容优先的信息结构。

### 英文简介

A pixel-inspired Hugo theme for personal blogs, focused on long-form readability, retro aesthetics, dark code blocks, and content-first layout.

## 开源协议

本主题建议采用：

`MIT License`

对应许可证文件已放置在主题目录中：

`themes/pixel-persona/LICENSE`

如果你把该主题单独拆分成一个独立 GitHub 仓库，请将该 `LICENSE` 文件放到仓库根目录。

## 建议开源范围

当前 `/root/HBlog` 是整站博客仓库，不只是主题仓库。

如果你要对外开源“主题”，建议只发布以下内容，而不是把整站文章内容一起放出去：

```text
pixel-persona/
├── archetypes/
├── assets/
├── i18n/
├── layouts/
├── static/
├── theme.toml
├── LICENSE
└── README.md
```

建议不要把以下内容作为主题仓库主体一起发布：

- 你的博客文章内容
- 站点级配置文件
- 与 `PaperMod` 兼容改造相关的旧文件
- 本地测试或个人部署配置
